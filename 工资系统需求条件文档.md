# 工资系统_Trae需求条件文档
## 一、系统总体架构要求
### 1.1 身份认证层
- 具体要求：基于「姓名」作为唯一登录账号，实现用户身份精准识别，登录姓名与工资表中姓名严格匹配，支持重名用户身份区分机制
- 优先级：**P0**
- 验收标准：姓名输入后完成唯一性验证，无此用户返回明确提示「无对应工资信息，无法登录」，重名用户可通过辅助信息区分
- 关联表格：工资表
- 技术约束：登录验证逻辑轻量化，响应延迟≤1s

### 1.2 数据交互层
- 具体要求：实现工资数据定向推送、签名文件（JPG/PNG/PDF）上传/存储/批量下载功能，全流程数据传输保持实时性与完整性
- 优先级：**P0**
- 验收标准：数据推送延迟≤3s，文件上传成功率100%，批量下载无文件丢失/损坏，支持断点续传
- 关联表格：工资表、汇总表
- 技术约束：文件存储按「姓名-时间戳」命名，建立唯一索引关联用户信息；数据传输采用HTTP协议

### 1.3 数据汇总层
- 具体要求：监听签名文件上传事件，自动整合用户工资信息与签名状态，按汇总表29个字段标准格式生成/更新汇总表，确保字段1:1精准映射
- 优先级：**P0**
- 验收标准：签名上传后≤10s完成汇总表更新，无字段错位/缺失/数据错误，支持手动触发汇总表刷新
- 关联表格：汇总表
- 技术约束：汇总表数据保留历史版本，支持按时间回溯查询

### 1.4 权限控制层
- 具体要求：严格划分普通用户与管理员权限，确保普通用户仅可访问本人工资信息、操作本人签名文件
- 优先级：**P0**
- 验收标准：普通用户无法访问其他用户数据，权限校验无漏洞，所有操作行为可追溯
- 关联表格：工资表、汇总表
- 技术约束：基于RBAC权限模型设计，权限配置支持后台灵活调整

## 二、后端功能要求
### 2.1 用户认证与数据推送模块
#### 2.1.1 姓名登录验证
- 具体要求：1.支持「姓名」作为唯一登录入参；2.登录时校验工资表中用户存在性；3.验证失败返回标准化错误提示；4.登录成功生成临时有效令牌（Token）
- 优先级：**P0**
- 验收标准：登录验证成功率100%，Token有效期可配置（默认24小时），过期后自动登出
- 关联表格：工资表
- 输出契约：登录结果、用户唯一标识、临时Token

#### 2.1.2 定向工资数据推送
- 具体要求：1.登录成功后按姓名匹配工资表对应完整数据；2.推送数据包含工资表30个字段所有信息；3.严格过滤其他用户数据，实现完全隔离；4.支持工资数据按需查询
- 优先级：**P0**
- 验收标准：推送数据仅含当前登录用户信息，字段完整、数值与原始工资表一致
- 关联表格：工资表
- 输入契约：用户姓名、有效Token；输出契约：本人工资完整结构化数据

### 2.2 签名文件管理模块
#### 2.2.1 文件上传接收
- 具体要求：1.支持JPG/PNG/PDF格式签名文件上传，单文件大小≤10M；2.上传时自动关联用户姓名与唯一标识，生成文件唯一ID；3.显示上传实时进度，完成后返回标准化确认信息；4.支持文件格式与大小校验，不符合则拦截并提示
- 优先级：**P0**
- 验收标准：指定格式文件上传成功率100%，文件与用户唯一关联，上传成功提示「签名文件提交完成，文件ID：XXX」
- 关联表格：汇总表
- 输入契约：用户唯一标识、签名文件流；输出契约：上传状态、文件唯一ID、存储路径、上传时间

#### 2.2.2 文件存储管理
- 具体要求：1.按「姓名-yyyyMMdd-HHmmss-文件ID」规则命名存储签名文件；2.建立文件索引表，关联用户信息、文件ID、存储路径、上传时间、文件状态；3.定期对文件进行完整性校验，损坏则及时标记并提示；4.敏感文件目录设置访问权限控制，禁止非法访问
- 优先级：**P0**
- 验收标准：文件命名规范，索引表关联准确，可通过姓名/文件ID快速查询，能有效检测文件损坏并生成告警
- 关联表格：汇总表
- 技术约束：文件采用本地+备份双存储机制，防止数据丢失

#### 2.2.3 批量下载功能
- 具体要求：1.仅向管理员开放批量下载入口；2.支持按「部门、入职日期、上传时间、姓名」多条件组合筛选；3.支持单文件下载与多文件ZIP打包下载；4.提供下载进度提示，完成后返回文件列表
- 优先级：**P1**
- 验收标准：可按条件精准筛选，打包下载无文件丢失，下载速度≥1M/s，支持同时下载≤50个文件
- 关联表格：汇总表
- 输入契约：管理员Token、筛选条件；输出契约：文件下载链接/打包文件流、文件列表

### 2.3 数据汇总与更新模块
#### 2.3.1 汇总表自动生成
- 具体要求：1.监听签名文件上传成功事件，触发汇总表生成/更新；2.按汇总表29个字段标准格式自动填充数据；3.自动补充「查询状态、查询次数、签收状态、最新签收时间、签收方式、查询数据编号」等状态字段；4.首次上传生成新记录，重复上传更新原有记录
- 优先级：**P0**
- 验收标准：上传成功后自动触发更新，状态字段按业务规则赋值（查询状态：已查询/未查询；签收状态：已签收/未签收），记录无重复
- 关联表格：工资表、汇总表
- 触发条件：签名文件上传成功事件

#### 2.3.2 数据字段精准映射
- 具体要求：实现工资表与汇总表核心字段1:1精准映射，重点对应：1.工资表「入职日期」→汇总表「入职日期」；2.工资表「基本工资-底薪」→汇总表「基本工资-底薪」；3.工资表「应扣款项」各子项→汇总表「应扣款项-XX」对应字段；4.工资表「实发工资/护理员绩效工资/应发工资」→汇总表同名字段；5.工资表「上门服务（小时）」→汇总表「上门服务（小时）」
- 优先级：**P0**
- 验收标准：所有字段映射100%准确，金额类数据保留2位小数，日期/数字字段格式统一，无转换错误
- 关联表格：工资表、汇总表
- 数据约束：金额类数据计算无精度损失，日期字段格式为「YYYY-MM-DD」

#### 2.3.3 汇总表实时更新与版本管理
- 具体要求：1.签名上传后≤10s完成汇总表更新；2.支持管理员手动触发全量刷新；3.更新后保留历史版本，按「更新时间」命名；4.支持按版本号/更新时间回溯查看、恢复历史数据
- 优先级：**P0**
- 验收标准：更新延迟≤10s，手动刷新生效，历史版本记录完整，可快速查询与恢复，无数据丢失
- 关联表格：汇总表
- 输出契约：汇总表最新数据、历史版本列表、版本更新日志

## 三、前端功能要求
### 3.1 用户登录与工资查看模块
- 具体要求：1.登录界面仅含「姓名输入框」+「登录按钮」，支持输入合法性校验；2.登录成功跳转工资查看页，以表格/卡片展示本人完整工资信息；3.工资信息按「基础信息、出勤信息、薪资构成、扣款明细」分类展示；4.提供「打印本人工资单」功能，格式规范；5.支持页面刷新，保持登录状态与数据展示
- 优先级：**P0**
- 验收标准：界面操作便捷，工资信息展示完整清晰，打印功能正常，页面刷新无异常，响应速度≤2s
- 关联表格：工资表
- 交互约束：输入框支持回车登录，登录按钮置灰时不可点击，加载状态显示动画提示

### 3.2 签名文件上传模块
- 具体要求：1.提供「点击上传」+「拖拽上传」两种方式，操作区域有明显提示；2.上传过程显示0%-100%进度条，支持取消上传；3.支持上传前文件预览，预览图尺寸适配页面；4.上传成功/失败显示文字+图标提示，失败提示具体原因；5.页面显示签名提交状态（未提交/已提交），已提交显示上传时间与文件预览
- 优先级：**P0**
- 验收标准：两种上传方式均生效，进度条实时更新，预览功能正常，提示清晰，状态展示准确，无重复上传问题
- 关联表格：汇总表
- 交互约束：拖拽文件至非上传区域无响应，取消上传后可重新选择文件

### 3.3 管理员操作界面
- 具体要求：1.提供独立管理员登录入口，与普通用户隔离；2.汇总表支持按「查询状态、签收状态、部门、姓名」筛选，分页展示（默认每页20条），支持列宽调整与排序；3.支持筛选后数据导出为Excel格式，包含所有字段；4.显示所有用户签名上传状态，支持单个/批量下载、在线预览；5.支持后台配置管理员账号，分配操作权限
- 优先级：**P1**
- 验收标准：功能操作流畅，筛选/分页/排序正常，导出文件数据完整，签名预览/下载无异常，权限配置生效
- 关联表格：汇总表
- 交互约束：自动记录操作日志，包含操作人、时间、内容

## 四、数据安全与权限控制要求
### 4.1 数据访问权限控制
- 具体要求：1.普通用户：仅可登录、查看本人工资、上传/查看/删除本人签名，无汇总表操作权限；2.管理员：可登录管理员界面、查看所有工资信息、导出汇总表、批量下载签名、配置权限、刷新汇总表；3.无权限操作时返回提示并拦截
- 优先级：**P0**
- 验收标准：权限划分清晰，无越权访问可能，操作日志可按操作人/时间/类型查询
- 关联表格：工资表、汇总表
- 技术约束：操作日志保存时间≥6个月，支持日志导出

### 4.2 数据传输安全
- 具体要求：1.全流程采用**HTTP协议**传输所有数据；2.工资敏感数据（实发工资、绩效工资）传输时进行二次加密处理；3.签名文件传输采用分片加密技术，分片大小≤1M；4.临时Token采用加密算法生成，防止伪造
- 优先级：**P0**
- 验收标准：Token无法伪造，敏感数据与签名文件传输无泄露，抓包无法获取明文信息
- 关联表格：工资表、汇总表
- 技术约束：采用国密加密算法进行二次加密

### 4.3 数据存储安全
- 具体要求：1.工资表原始数据每日自动备份，备份文件存储在独立服务器，备份周期≥30天；2.签名文件存储目录设置严格访问权限，仅授权服务可访问；3.敏感薪资数据（实发工资、应发工资）存储时脱敏处理，仅展示时还原；4.数据库设置访问密码，支持数据库操作日志
- 优先级：**P0**
- 验收标准：备份文件可正常恢复，文件目录无非法访问记录，敏感数据无明文存储，数据库操作可追溯
- 关联表格：工资表、汇总表
- 技术约束：数据库采用主从复制架构；**数据库密码无需定期更新**

## 五、数据格式标准化要求
### 5.1 工资表数据格式规范
- 具体要求：统一所有字段数据类型、格式与精度，核心规范：1.基础信息类（序号/部门/姓名/岗位）：文本类型；2.日期类（入职日期）：日期类型，格式「YYYY-MM-DD」；3.出勤类（应出勤/实际出勤天数）：数字类型，保留1位小数；上门服务小时：数字类型，保留0位小数；4.薪资/扣款类：数字类型，保留2位小数；5.是否代扣社保：布尔类型，取值「是/否」
- 优先级：**P0**
- 验收标准：所有字段格式符合规范，数据精度无丢失，类型匹配无错误
- 关联表格：工资表
- 数据约束：金额类≥0，实际出勤天数≤应出勤天数，上门服务小时≥0

### 5.2 汇总表数据格式规范
- 具体要求：严格遵循29个字段格式，在工资表规范基础上补充：1.查询/签收状态：文本类型，固定取值「已查询/未查询」「已签收/未签收」；2.最新签收时间：时间戳类型，格式「YYYY-MM-DD HH:MM:SS」；3.签名图片：文本类型，存储文件唯一ID/访问路径；4.姓名/序号/部门：与工资表格式完全一致；5.查询次数：数字类型，默认0、取值≥0
- 优先级：**P0**
- 验收标准：所有字段格式100%合规，与工资表关联字段一致，固定取值字段无自定义值
- 关联表格：汇总表
- 数据约束：最新签收时间≥入职日期，查询次数随工资查看操作自动累加

### 5.3 数据格式校验与转换
- 具体要求：1.后端接收/生成数据时自动格式校验，不符合则拦截并提示；2.支持非标准格式自动转换为标准化格式；3.前端按标准化格式渲染展示；4.数据导入/导出时保持标准化格式，无错乱
- 优先级：**P0**
- 验收标准：格式校验成功率100%，非标准格式可自动转换，展示/导入/导出格式统一
- 关联表格：工资表、汇总表
- 输出契约：校验结果、转换后数据、错误信息提示
