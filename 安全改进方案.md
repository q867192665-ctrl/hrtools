# 数据传输安全改进方案

## 当前安全问题

### 1. 使用HTTP明文传输（严重）
- **风险等级**：严重
- **问题描述**：所有数据（密码、Token、工资信息等）以明文形式在网络中传输
- **潜在威胁**：
  - 中间人攻击（MITM）
  - 数据窃听
  - 数据篡改
  - 身份冒充

### 2. 密码明文传输（严重）
- **风险等级**：严重
- **问题描述**：登录时密码以明文形式发送
- **潜在威胁**：
  - 密码被截获
  - 账号被盗用
  - 用户隐私泄露

### 3. Token明文传输（高危）
- **风险等级**：高危
- **问题描述**：身份验证Token在HTTP Header中明文传输
- **潜在威胁**：
  - Token被盗用
  - 会话劫持
  - 未授权访问

## 解决方案

### 方案一：配置HTTPS（推荐）

#### 1. 获取SSL证书

**选项A：使用免费证书（Let's Encrypt）**
```bash
# 安装Certbot
sudo apt-get update
sudo apt-get install certbot

# 获取证书（需要域名）
sudo certbot certonly --standalone -d yourdomain.com
```

**选项B：使用自签名证书（仅用于测试）**
```bash
# 生成私钥
openssl genrsa -out server.key 2048

# 生成证书签名请求
openssl req -new -key server.key -out server.csr

# 生成自签名证书
openssl x509 -req -days 365 -in server.csr -signkey server.key -out server.crt
```

#### 2. 配置Flask使用HTTPS

修改 `backend/app.py`：

```python
if __name__ == '__main__':
    # 使用HTTPS
    app.run(
        host='::',  # IPv6
        port=5000,
        ssl_context=('../server.crt', '../server.key'),  # 证书和私钥路径
        debug=True
    )
```

#### 3. 修改Android APP使用HTTPS

修改所有Activity中的 `buildUrl()` 函数：

```kotlin
private fun buildUrl(address: String): String {
    val trimmed = address.trim()
    return if (trimmed.contains(":") && !trimmed.startsWith("[")) {
        "https://[$trimmed]:5000"  // 改为 https
    } else if (trimmed.startsWith("[")) {
        "https://$trimmed"  // 改为 https
    } else {
        "https://$trimmed:5000"  // 改为 https
    }
}
```

#### 4. 配置OkHttpClient信任证书

如果是自签名证书，需要配置OkHttpClient信任证书：

```kotlin
private fun getUnsafeOkHttpClient(): OkHttpClient {
    try {
        val trustAllCerts = arrayOf<TrustManager>(object : X509TrustManager {
            override fun checkClientTrusted(chain: Array<java.security.cert.X509Certificate>, authType: String) {}
            override fun checkServerTrusted(chain: Array<java.security.cert.X509Certificate>, authType: String) {}
            override fun getAcceptedIssuers(): Array<java.security.cert.X509Certificate> = arrayOf()
        })

        val sslContext = SSLContext.getInstance("SSL")
        sslContext.init(null, trustAllCerts, java.security.SecureRandom())
        
        val builder = OkHttpClient.Builder()
        builder.sslSocketFactory(sslContext.socketFactory, trustAllCerts[0] as X509TrustManager)
        builder.hostnameVerifier { _, _ -> true }
        
        return builder.build()
    } catch (e: Exception) {
        throw RuntimeException(e)
    }
}
```

```kotlin
import java.security.MessageDigest

private fun hashPassword(password: String): String {
    val md = MessageDigest.getInstance("SHA-256")
    val hashedPassword = md.digest(password.toByteArray())
        .joinToString("") { "%02x".format(it) }
    return hashedPassword
}
```

```python
def login(self, name, password):
    """
    用户登录验证
    password: 可以是明文或SHA256加密后的密码
    """
    conn = None
    try:
        conn = self.get_db_connection()
        cursor = conn.cursor()
        
        cursor.execute(
            "SELECT 姓名, 密码, role FROM users WHERE 姓名 = ?",
            (name,)
        )
        user = cursor.fetchone()
        
        if not user:
            return {
                'success': False,
                'error': '用户不存在'
            }
        
        # 尝试验证：可能是明文或SHA256
        stored_password = user['密码']
        
        # 方式1：直接验证（如果前端传的是明文）
        if self.verify_password(stored_password, password):
            pass  # 验证成功
        # 方式2：SHA256验证（如果前端传的是加密后的密码）
        elif self.verify_sha256_password(stored_password, password):
            pass  # 验证成功
        else:
            return {
                'success': False,
                'error': '密码错误'
            }
```

## 其他安全建议

### 1. Token安全
- Token有效期设置为24小时
- Token使用SHA256哈希存储
- 登出时清除Token

### 2. 密码安全
- 密码使用bcrypt加密存储
- 强制密码复杂度要求
- 定期提醒用户修改密码

### 3. 设备绑定
- 已实现设备ID绑定
- 记录异常登录尝试
- 支持设备解绑

### 4. 日志审计
- 记录所有登录尝试
- 记录敏感操作
- 支持日志查询

## 合规性说明

根据《网络安全法》和《个人信息保护法》：
- 敏感个人信息必须加密传输
- 应当采取技术措施保护个人信息安全
- 发生安全事件应当及时告知用户
