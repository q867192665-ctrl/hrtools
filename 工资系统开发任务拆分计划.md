# 工资系统开发任务拆分计划

## 📋 任务概览

本文档基于《工资系统需求条件文档》，按前后端拆分可独立执行的原子开发任务，优先实现所有P0优先级功能。

### 技术约束
- ✅ HTTP协议传输所有数据
- ✅ 数据库密码无需定期更新
- ✅ 工资表与汇总表字段1:1精准映射
- ✅ 响应延迟要求：登录≤1s，数据推送≤3s，汇总更新≤10s

## 🎯 P0优先级任务清单

### 阶段一：数据库设计与初始化（前置任务）

#### 任务1.1：设计数据库表结构
**优先级**：P0  
**类型**：后端-数据库  
**预估工时**：2小时  

**任务描述**：
设计并创建4个核心数据表，确保字段完整性和关联关系正确。

**输入契约**：
- 工资表字段规范（30个字段）
- 汇总表字段规范（29个字段）
- 用户权限需求
- 签名文件管理需求

**输出契约**：
- 完整的SQL建表脚本
- 数据库ER图文档
- 字段说明文档

**数据表设计**：

1. **工资表 (salary_table)**
```sql
CREATE TABLE salary_table (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    序号 TEXT NOT NULL,
    部门 TEXT NOT NULL,
    姓名 TEXT NOT NULL UNIQUE,
    岗位 TEXT,
    入职日期 DATE NOT NULL,
    应出勤天数 DECIMAL(4,1),
    实际出勤天数 DECIMAL(4,1),
    上门服务小时 INTEGER,
    基本工资底薪 DECIMAL(10,2),
    基本工资绩效 DECIMAL(10,2),
    基本工资合计 DECIMAL(10,2),
    岗位工资 DECIMAL(10,2),
    护理员绩效工资 DECIMAL(10,2),
    应发工资 DECIMAL(10,2),
    应扣款项社保 DECIMAL(10,2),
    应扣款项公积金 DECIMAL(10,2),
    应扣款项个税 DECIMAL(10,2),
    应扣款项其他 DECIMAL(10,2),
    应扣款项合计 DECIMAL(10,2),
    实发工资 DECIMAL(10,2),
    是否代扣社保 BOOLEAN,
    备注 TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

2. **汇总表 (summary_table)**
```sql
CREATE TABLE summary_table (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    序号 TEXT NOT NULL,
    部门 TEXT NOT NULL,
    姓名 TEXT NOT NULL,
    入职日期 DATE NOT NULL,
    应出勤天数 DECIMAL(4,1),
    实际出勤天数 DECIMAL(4,1),
    上门服务小时 INTEGER,
    基本工资底薪 DECIMAL(10,2),
    基本工资绩效 DECIMAL(10,2),
    基本工资合计 DECIMAL(10,2),
    岗位工资 DECIMAL(10,2),
    护理员绩效工资 DECIMAL(10,2),
    应发工资 DECIMAL(10,2),
    应扣款项社保 DECIMAL(10,2),
    应扣款项公积金 DECIMAL(10,2),
    应扣款项个税 DECIMAL(10,2),
    应扣款项其他 DECIMAL(10,2),
    应扣款项合计 DECIMAL(10,2),
    实发工资 DECIMAL(10,2),
    是否代扣社保 BOOLEAN,
    查询状态 TEXT CHECK(查询状态 IN ('已查询', '未查询')),
    查询次数 INTEGER DEFAULT 0,
    签收状态 TEXT CHECK(签收状态 IN ('已签收', '未签收')),
    最新签收时间 TIMESTAMP,
    签收方式 TEXT,
    签名图片 TEXT,
    查询数据编号 TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    version INTEGER DEFAULT 1
);
```

3. **用户表 (users)**
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    姓名 TEXT NOT NULL UNIQUE,
    role TEXT CHECK(role IN ('user', 'admin')) DEFAULT 'user',
    token TEXT,
    token_expire_time TIMESTAMP,
    last_login_time TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

4. **签名文件表 (signature_files)**
```sql
CREATE TABLE signature_files (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    文件ID TEXT NOT NULL UNIQUE,
    用户姓名 TEXT NOT NULL,
    文件名 TEXT NOT NULL,
    存储路径 TEXT NOT NULL,
    文件格式 TEXT CHECK(文件格式 IN ('JPG', 'PNG', 'PDF')),
    文件大小 INTEGER,
    上传时间 TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    文件状态 TEXT CHECK(文件状态 IN ('正常', '损坏', '已删除')),
    FOREIGN KEY (用户姓名) REFERENCES users(姓名)
);
```

**验收标准**：
- ✅ 所有表创建成功，无SQL错误
- ✅ 字段类型、约束符合需求文档规范
- ✅ 外键关联正确
- ✅ 索引创建合理（姓名、文件ID等）

**依赖关系**：无

### 阶段二：后端核心功能开发

#### 任务2.1：实现姓名登录验证API
**优先级**：P0  
**类型**：后端-API  
**预估工时**：3小时  

**任务描述**：
实现基于姓名的唯一登录验证，生成临时Token。

**输入契约**：
```json
{
  "姓名": "张三"
}
```

**输出契约**：
```json
{
  "success": true,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "姓名": "张三",
    "role": "user",
    "expire_time": "2024-01-02 10:00:00"
  }
}
```

**API接口**：
- 路径：`POST /api/login`
- 方法：HTTP POST
- 协议：HTTP

**实现要点**：
1. 校验姓名在工资表中是否存在
2. 不存在返回：「无对应工资信息，无法登录」
3. 存在则生成Token（有效期24小时）
4. Token采用加密算法生成（防止伪造）
5. 更新用户最后登录时间

**验收标准**：
- ✅ 姓名验证成功率100%
- ✅ Token有效期可配置（默认24小时）
- ✅ 响应延迟≤1s
- ✅ 无此用户返回明确提示
- ✅ Token无法伪造

**依赖关系**：任务1.1（数据库表创建）

#### 任务2.3：实现签名文件上传API
**优先级**：P0  
**类型**：后端-API  
**预估工时**：5小时  

**任务描述**：
支持JPG/PNG/PDF格式签名文件上传，自动关联用户信息。

**输入契约**：
```json
{
  "headers": {
    "Authorization": "Bearer {token}",
    "Content-Type": "multipart/form-data"
  },
  "body": {
    "file": "<binary>",
    "format": "JPG|PNG|PDF"
  }
}
```

**输出契约**：
```json
{
  "success": true,
  "message": "签名文件提交完成，文件ID：SF20240101120000",
  "data": {
    "文件ID": "SF20240101120000",
    "存储路径": "/signatures/张三-20240101-120000-SF20240101120000.jpg",
    "上传时间": "2024-01-01 12:00:00",
    "文件大小": 102400
  }
}
```

**API接口**：
- 路径：`POST /api/signature/upload`
- 方法：HTTP POST
- 协议：HTTP

**实现要点**：
1. 验证Token有效性
2. 校验文件格式（仅JPG/PNG/PDF）
3. 校验文件大小（≤10M）
4. 生成文件唯一ID（格式：SF+时间戳）
5. 按「姓名-yyyyMMdd-HHmmss-文件ID」命名存储
6. 建立文件索引记录
7. 触发汇总表更新事件

**验收标准**：
- ✅ 指定格式文件上传成功率100%
- ✅ 文件与用户唯一关联
- ✅ 文件命名规范正确
- ✅ 上传成功返回文件ID
- ✅ 不符合格式/大小的文件被拦截

**依赖关系**：任务1.1（数据库表创建）

#### 任务4.2：实现数据传输安全
**优先级**：P0  
**类型**：后端-安全  
**预估工时**：4小时  

**任务描述**：
实现HTTP协议下的数据传输安全保护。

**安全措施**：
1. 全流程采用HTTP协议传输
2. 敏感数据（实发工资、绩效工资）二次加密
3. 签名文件分片加密传输（分片≤1M）
4. Token加密生成，防止伪造

**实现要点**：
1. 采用国密加密算法进行二次加密
2. 前端加密后传输，后端解密
3. 签名文件分片上传
4. Token采用JWT格式，包含签名验证

**验收标准**：
- ✅ Token无法伪造
- ✅ 敏感数据传输无泄露
- ✅ 签名文件传输无泄露
- ✅ 抓包无法获取明文信息

**依赖关系**：任务2.1（登录验证API）
